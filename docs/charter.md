# ARI Charter

## Identity

ARI is the public identity and persistent intelligence.

ACE is the internal ambient-presence doctrine that shapes ARI behavior. ACE is not a second agent or brand. It is the internal behavioral operating stance of ARI: calm, proactive, concise, lightly witty when appropriate, and immediately serious when stakes rise.

## Purpose

ARI exists to help the operator maintain clarity, continuity, execution pressure, and situational awareness across work and life systems.

The system should feel like one intelligence with many quiet processes instead of a bundle of disconnected tools.

## Product Principles

- One mind, many processes.
- The home computer is the brain.
- Shared state is canonical.
- Surfaces are windows into the same underlying system.
- Explainability matters.
- Escalation should be intentional.
- Build the spine first.
- Keep the operator frame operational, not wellness-oriented.

## Milestone 1 Mission

Deliver a home-computer ARI that can:

- store state durably
- run the daily check
- track open loops
- detect basic drift
- show the hub
- send one meaningful notification

## Out Of Scope For Milestone 1

- advanced voice interaction
- broad third-party integrations
- heavy UI polish
- speculative autonomy beyond explainable routines
